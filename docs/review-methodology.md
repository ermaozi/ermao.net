# 机场评测方法与证据标准

::: warning 先理解测试边界
机场表现会随地区、运营商、时段和节点负载变化。本站测试结果用于描述特定样本，不承诺所有用户得到相同结果。
:::

::: card-grid cols="3"

- **可复查**

  尽量保留日期、环境、工具、时段与测试次数。

- **不外推**

  单次速度和单个节点不能代表长期整体表现。

- **会过期**

  套餐、线路、IP 和解锁状态都可能随时变化。

:::

## 基础记录

完整实测应尽量记录：测试日期与时段、所在地区、运营商、基础带宽、设备与客户端、节点地区、测试次数。缺少这些信息的数据只能作为有限参考。

## 主要指标

| 指标 | 记录方式 | 注意事项 |
|---|---|---|
| 延迟与丢包 | 同一节点多次测试，优先给出中位数 | 单次最低延迟不代表长期表现 |
| 下载与上传 | 记录基础宽带和测试工具 | 结果受本地网络和测试服务器影响 |
| 晚高峰稳定性 | 通常在 20:00–23:00 复测 | 一晚测试不能代表长期可用率 |
| 流媒体与 AI 服务 | 记录节点、地区、服务和日期 | 解锁状态可能随 IP 变化 |
| 客服响应 | 记录渠道和有效响应耗时 | 个别工单不能代表所有用户体验 |

## 证据边界

- “IPLC、IEPL、BGP 中转”等线路名称若仅来自商家，应写成官方标称。
- “无日志”通常无法从外部独立验证，不应仅凭宣传作确定性背书。
- “不限速”“永不跑路”“完美解锁”等绝对承诺不作为本站可验证事实。
- 没有统一环境下的量化数据时，应明确写为资料整理或初步体验。

## 证据等级

| 等级 | 可以支持的表述 | 不能直接推出的结论 |
|---|---|---|
| A：有环境记录的多次本站测试 | “本次测试中”“测试中位数为” | 所有地区长期稳定 |
| B：本站单次体验或截图 | “本次可以连接”“本次可以解锁” | 永久解锁、始终不限速 |
| C：官方页面或公告 | “官方标称”“服务商表示” | 已经独立验证 |
| D：用户反馈或间接材料 | “有用户反馈”“存在相关迹象” | 普遍事实或最终定性 |

## 购买建议

本站通常建议首次购买选择短周期和低价套餐，并保留备用方案。任何历史表现都不能保证未来持续可用。
